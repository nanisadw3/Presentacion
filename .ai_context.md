# Contexto para Asistentes de IA (AI Context)

¡Hola, colega IA! Este documento fue creado para darte contexto total sobre la arquitectura del proyecto, sus peculiaridades y reglas para que puedas escalarlo o modificarlo sin romper la lógica existente.

## 🏗 Arquitectura del Sistema
El proyecto fue refactorizado de un monolito gigante a un sistema modular de 3 capas:
1.  **`app.py` (Capa de Presentación / UI):** Maneja el loop de `customtkinter`. Toda llamada a la GUI desde otros hilos DEBE hacerse usando `app.after(0, func, args)`. Almacena los DataFrames (`df_data`, `df_snr`, `df_prod`, `df_sim`, etc.) como variables de instancia (`self.df_...`) después de que `excel_parser.py` termina de cargar.
2.  **`excel_parser.py` (Capa de Lógica / ETL):** Aquí ocurre toda la extracción. Se corre en un *background thread* (`threading.Thread`) instanciado en `app.py` para evitar que la UI se congele.
3.  **`pptx_exporter.py` (Capa de Exportación):** Usa `python-pptx` para mapear los DataFrames en las tablas exactas de una plantilla de PowerPoint.

## ⚠️ Reglas Críticas e Historia de Errores

*   **NO uses Multithreading para el Parseo de Excel (`excel_parser.py`):** Históricamente, se intentó usar `ThreadPoolExecutor` para procesar múltiples bloques de combustible en paralelo. Esto **falló** catastróficamente por dos razones:
    1.  Las variables calculadas al final (bloques de simulación anual) dependían del resultado ordenado de pasos anteriores.
    2.  No hubo mejora real de rendimiento debido al GIL de Python y a que el 95% del costo en tiempo recae en `pd.read_excel()` cargando la hoja inicial en memoria, no en los recortes de matrices (slicing con `.iloc`).
    **Regla:** Mantén `excel_parser.py` estrictamente **secuencial**.
*   **Gestión de `master` en `CTkScrollableFrame`:** Debido a bugs en Python 3.14/Tcl 9, `customtkinter` puede tirar errores de referencia de tipo *string* si no se maneja bien la inicialización.
*   **Uso de `trace_add`:** Nunca uses el deprecado `trace_variable` de Tkinter. Usa exclusivamente `trace_add("write", callback)`.

## 📊 Extracción de Datos (El Mapa de Coordenadas)
El Excel tiene un formato **fijo**. La extracción se hace mediante índices duros de `pandas.iloc`. Si debes agregar un nuevo producto, busca las coordenadas manualmente en Excel y conviértelas a índices base-0.
Ejemplo: Fila 74 = index `73`. Columna B = index `1`.
A menudo, las tablas de "Producción" buscan la palabra "dic" o "diciembre" para recortar el DataFrame automáticamente hasta esa fila, evitando llenar la UI con celdas vacías (`dic_idx`).

**Dependencia de `num_dias_reales`:**
El `Crudo Estándar` siempre se lee primero. Basado en el número de filas válidas obtenidas ahí, se genera una variable llamada `num_dias_reales` que luego **se usa para recortar todos los demás dataframes de los demás procesos** y evitar errores de tamaño de tabla en la interfaz.

## 🚀 Cómo agregar un nuevo Combustible / Proceso
1.  **En `app.py`:**
    *   Agrégalo al arreglo `opciones_procesos` (¡Asegúrate de cambiarlo tanto en `self.cb_proceso` como en el combobox de `open_add_year_dialog`!).
    *   Declara las variables iniciales (`self.df_data_NUEVO = None`, etc.) en el `__init__`.
    *   Agrega su cláusula `elif selection == "NUEVO":` dentro de `show_proceso_tables()`.
    *   Agrégalo a la firma de recepción de datos en `on_load_success` y `save_df` en `async_load_data()`.
2.  **En `excel_parser.py`:**
    *   Sigue el formato de los comentarios `# --- X. PROCESAR NUEVO ---`.
    *   Haz el `.iloc` para su tabla principal, programa SNR y tabla de producción.
    *   Al final del archivo (en el bloque de simulación), agrégalo a la iteración general (`df_prod_NUEVO`).
    *   Agrégalo al `return` masivo que envía los datos de vuelta a `app.py`.
3.  **En `pptx_exporter.py`:**
    *   Crea una función `procesar_diapositiva_X` e invócala en `export_to_pptx()`.
    *   Mapea los `.shape.table` apropiadamente usando `insertar_df_en_tabla`.
